

#### 	**1**  在学习git之前，我们要先了解什么是git，其是一个分布式版本的控制系统，帮助你管理文件的工具为代码设计而生，而且也可以硬座很多文本文件。它的核心内容包括：

|          概念          |                             内容                             |
| :--------------------: | :----------------------------------------------------------: |
|    **仓库（Repo）**    |            被git管理的项目文件夹，包含很多子目录             |
|   **提交（Commit）**   |                    把内容提交到本地仓库。                    |
|   **分支（Branch）**   | 顾名思义就是在一个主分支（Mian）下的其他 分支，我们在其他分支上写入内容，最后进行合并到主分支上。 |
|  **暂存区（Stage）**   |         准备提交的内容，还没有提交到远程仓库的文件。         |
| **远程仓库（Remote）** | 把在暂存区的内容提交到远程仓库，同时可以设置远程仓库的权限，分为私人和公开。 |

​	在此之后我们还要明白Git和Github的区别，Git是工具本身，而Github是提供Git的仓库的托管的网站。

​	总结就是，GIt是一个可以安全的记录每一步的修改和提交，同时可以回到过去版本，并且可以和其他人无缝协作的编写代码的工具。

### 2.2分支的概念

​	几乎所有的版本控制系统都以某种形式支持分支。 使用分支意味着你可以把你的工作从开发主线上分离 开来进行重大的Bug修改、开发新的功能，以免影响开发主线。

##### 2.2.1查看本地分支：

*命令：git branch

##### 2.2.2创建本地分支

*命令：git branch 分支名

##### 2.2.3切换分支（checkout）

*命令：git checkout 分支名

##### 2.2.4创建并切换分支

*命令：git checkout -b 分支名（建议使用比较快速）

##### 2.2.5合并分支（merge）

一个分支上的提交合并到另一个分支

*命令：git merge 分支名称

##### 2.2.6删除分支

不可以删除当前的分支，只能删除其他分支

*命令：git branch -d 删除分支时，要做各种检查

*命令：git branch -D不做任何检查，强制删除（不建议使用）

### 2.3分支的命名

#### github的主流分支命名

##### 2.3.1主干分支

main（推荐使用，github默认使用）

##### 2.3.2集成分支

通常没有，直接可以从main拉去功能分支

##### 2.3.3功能分支

feature/<功能简述>（例如feature/user-login-api)

##### 2.3.4Bug修复分支

hotfix/<问题简述>(从main拉取)

##### 2.3.4发布分支

release/<版本号>(可选择，用于正式的发布)

##### 2.3.5文档等分支

docs/<描述>,chore<描述>等

##### 	对于github分支，这是目前互联网公司最流行的“轻量级”模型，核心思想是**主干开发，持续交付**。它通常只有`main`一个长期分支，任何新功能或修复都从`main`拉取一个短期的`feature`或`hotfix`分支，完成后通过拉取请求（PR）合并回`main`，可快速部署。

​	在Repository name一栏，我们可以填入自己存储库的名字，在Description一栏可以填入自己对存储库的描述，例如“这是一个学习git的存储库，在choose visibility 我们可以选择public和private，选择那个都可以，取决于你是想把你的库公开还是只是自己试用，其他选项吗则不用管，点击Create repository，即可创建一个自己的存储库。

##### 3.1.1添加远程仓库

此操作是先初始化本地库，然后与已创建的远程库进行对接使用的。

*命令：git remote add<远端名称><仓库路径>

#远端名称，默认是origin，取决于远端服务器的设置

#仓库路径，从远端服务器获取此URL

#例如：git@github.com:ulearn-student/test.git

##### 3.1.2查看远程仓库

*命令：git remote

##### 3.1.3推送到远程仓库

*命令：git push origin main

##### 3.1.4查看本地分支和远程分支的关联

*命令：git branch -vv

##### 3.1.5从远程仓库克隆

git clone <git@github.com:ulearn-student/test.git>

##### 3.1.6从远程仓库push和pull

#远程分支和本地分支一样，我们可以进行merge操作，只是需要先把远程仓库里的更新都下载到本地，在进行操作。

*抓取命令：git fetch <remote name > <branch name>

#抓取就是将仓库的更新都抓取到本地，但是不会进行合并，如果不指定远端名称和分支名，则抓取所有的分支。

*拉取命令：git pull <remote name> <branch name>

#拉取就是将远端仓库的修改到本地并进行自动合并，等同于fetch+merge,如果不指定远端名称和分支名，则抓取所有并更新当前分支。
